# PO communication (Jira comments on Epics)

When escalating an open question to the PO, do **not** present options as a flat A/B/C menu — the PO will tend to pick the most defensive option without seeing its cost. Instead:

1. State the question and our recommended default.
2. Explicitly name the dev cost of the alternative ("construir X es un módulo aparte de varios días").
3. Ask for consent or pushback ("¿Te encaja, o crees que el riesgo justifica la ampliación?").

Apply across all `/speckit-atlassian-sync-push` comments and any manual PO outreach for Afianza Epics.
