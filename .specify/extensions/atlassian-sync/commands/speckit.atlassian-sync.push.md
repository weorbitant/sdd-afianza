# Atlassian Sync — Push to Jira

Reads the active feature's `spec.md` and creates Jira Stories under an existing Epic.
**Never modifies an existing Epic** — description, summary, and fields are left untouched.
If the Epic has Open Questions (`## Open Questions` section in spec.md), adds them as a
single comment on the Epic so the PO and team can see them without altering the Epic body.

```
Epic (existing — untouched)
├── Comment  ←  open questions for PO (if any)
└── Story    ←  each User Story in spec.md (created only if not already linked)
```

## Usage

```
/speckit-atlassian-sync-push [epic-key]
```

- `epic-key` (optional): Jira key of an **existing** Epic (e.g. `DEVPT-518`).
  If omitted, a new Epic is created automatically from `spec.md` title.

---

## Execution Steps

### 1. Load configuration

Read `.specify/extensions/atlassian-sync/config/atlassian.yml` for defaults
(`cloudId`, `projectKey`, `issueTypes`, `epicLinkField`).
If `specs/<feature>/atlassian.yml` exists, merge it on top (feature overrides win).

### 2. Check prerequisites

Run `.specify/scripts/bash/check-prerequisites.sh --json` from the repo root.
Parse JSON for `FEATURE_DIR`. Abort if `spec.md` is missing.

Verify the Atlassian MCP connection. If not available, abort:
> "Atlassian MCP is not connected. Run `/mcp` and reconnect before retrying."

### 3. Resolve Epic

**If `epic-key` is provided:**
- Validate via `mcp__atlassian__getJiraIssue { issueIdOrKey: "<epic-key>" }`.
- Confirm issue type is Epic. If not, abort.
- Store as `EPIC_KEY`.
- **Do NOT modify** the Epic's summary, description, or any other field.

**If `epic-key` is omitted:**
- Extract Epic name from `spec.md` H1 (`# Feature Specification: <name>` → `<name>`).
- Extract Epic description from the first paragraph after the title or the Summary section.
- Create a new Epic via `mcp__atlassian__createJiraIssue`:
  ```json
  { "summary": "<name>", "description": "<description>", "issueType": "Epic" }
  ```
- Store returned key as `EPIC_KEY`.
- Report: `✅ Epic created: <EPIC_KEY>`

### 4. Post Open Questions as Epic comment (if any)

Parse `spec.md` for a `## Open Questions` section.
If found and contains at least one `OQ-` row:

- Format the questions as a clean Markdown comment:
  ```
  ## 🔓 Open Questions — Pending PO Decision

  These questions were identified during refinement and require a decision
  before the technical planning phase can begin.

  | # | Area | Question | Impact |
  |---|------|----------|--------|
  | OQ-001 | ... | ... | ... |
  ...

  _Generated by Spec Kit on <date>_
  ```
- Post via `mcp__atlassian__addCommentToJiraIssue { issueIdOrKey: "<EPIC_KEY>", body: "..." }`.
- Report: `💬 Open questions posted as comment on <EPIC_KEY>`.
- If section is absent or empty, skip silently.

### 5. Fetch existing Stories to avoid duplicates

Call `mcp__atlassian__searchJiraIssuesUsingJql`:
```
project = <projectKey> AND "Epic Link" = <EPIC_KEY> AND issuetype = Story
```
Collect existing story summaries (lowercased) into `EXISTING_SUMMARIES`.

### 6. Create Stories

For each `### User Story N` block in `spec.md`:

Extract:
- **Summary**: the story title (text after `### User Story N —`)
- **Description**: full functional content — narrative paragraph, **Why this priority**, **Independent Test**, and all **Acceptance Scenarios** (Given/When/Then)

Skip if a story with the same summary (case-insensitive) already exists in `EXISTING_SUMMARIES`.

For each story to create, call `mcp__atlassian__createJiraIssue`:
```json
{
  "cloudId": "<cloudId>",
  "projectKey": "<projectKey>",
  "summary": "<story title>",
  "description": "<full story content as markdown>",
  "issueType": "Story",
  "<epicLinkField>": "<EPIC_KEY>"
}
```

Report each: `✅ Story created: <KEY> — <title>` or `↷ Skipped (already exists): <title>`.

### 7. Output summary

Print a results table:

| Result | Key | Summary |
|--------|-----|---------|
| ✅ Created | DEVPT-NNN | Story title |
| ↷ Skipped | — | Story title (already exists) |
| 💬 Comment | EPIC_KEY | Open questions posted |

Finish: `X Stories created, Y skipped. Epic <EPIC_KEY> unchanged.`

---

## Error Handling

- MCP not connected → abort with reconnect instructions.
- `spec.md` missing → abort with path hint.
- Epic key invalid or not an Epic → abort.
- Epic creation fails (no key provided) → abort.
- Individual Story creation fails → log error, continue with remaining stories.
- Comment posting fails → log warning, continue (non-blocking).
